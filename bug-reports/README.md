# Bug Reports

This folder contains sample defect reports demonstrating professional bug documentation practices.

## Format Used

Each bug report includes:
- **Severity** — Technical impact (Critical / High / Medium / Low)
- **Priority** — Business urgency (High / Medium / Low)
- **Status** — Defect lifecycle stage (New / Open / In Progress / Fixed / Closed)
- **Steps to Reproduce** — Numbered, specific, independently reproducible
- **Expected vs. Actual** — Clear comparison of intended vs. observed behavior
- **Evidence** — Screenshots / screen recordings *(placeholders in samples)*
- **Notes** — Root cause hypothesis and suggested fix direction
